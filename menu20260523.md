## 1. 技术方案

### 1.1 Widget 选择

使用 **`Gtk.PopoverMenuBar`**（GTK 4.10+），这是 GTK4 推荐的传统菜单栏替代方案：

- 横向排列的按钮栏，点击弹出下拉菜单
- 与 GNOME HIG 兼容，不引入传统 `Gtk.MenuBar`
- 通过 `Gio.MenuModel` 驱动的声明式菜单定义

### 1.2 架构

```
Gio.Menu (menu model tree)
  └─ Gtk.PopoverMenuBar.new_from_model(model)
       └─ 挂载到 vbox 顶部（HeaderBar 下方、Toolbar 上方）

GAction (动作系统)
  ├─ Gio.SimpleAction.new(name, None)
  ├─ self.add_action(action)  → 注册为 win.* 域动作
  └─ 菜单通过 'win.action_name' 字符串引用
```

### 1.3 动作注册

```python
action_defs = [
    ('open', self._on_open_archive),        # 已实现
    ('close', self._on_close_archive),      # 已实现
    ('password', self._on_not_implemented), # 占位
    ...
]
for name, cb in action_defs:
    action = Gio.SimpleAction.new(name, None)
    action.connect('activate', lambda a, p, cb=cb: cb())
    self.add_action(action)
```

### 1.4 菜单模型构建

使用辅助函数简化 `Gio.Menu` 构建：

```python
def sub(*items):
    m = Gio.Menu()
    for item in items:
        if item is None:                     # → 分隔线
            m.append_section(None, Gio.Menu())
        elif isinstance(item, tuple):        # → 菜单项
            m.append(*item)
    return m
```
